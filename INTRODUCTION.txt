[h1]RimMind - Core[/h1]
RimMind 套件的核心基础设施，提供 LLM 客户端、异步请求队列和游戏上下文构建器，是所有 RimMind 子模组的前置依赖。

[h2] RimMind 是什么 [/h2]
RimMind 是一套 AI 驱动的 RimWorld 模组套件，通过接入大语言模型 (LLM)，让殖民者拥有人格、记忆、对话和自主决策能力。

[h2] 核心能力 [/h2]
[b] 智能上下文构建 [/b] - 自动采集游戏状态：殖民者心情、健康、技能、装备、地图财富、威胁等级、季节天气等，为 AI 决策提供完整情报。
[b] 异步请求队列 [/b] - 智能管理 LLM 请求，避免阻塞游戏主线程，支持请求优先级和冷却控制。
[b] 模块化 API 设计 [/b] - 其他 RimMind 组件通过统一接口调用，无需关心底层实现细节。
[b] 上下文过滤器 [/b] - 28+ 个可配置选项，精确控制注入 Prompt 的信息量，节省 Token 开销。
[b] 调试工具集 [/b] - 内置 AI 请求日志窗口，实时查看每次 AI 调用的输入输出，方便排查问题。

[h2] 技术亮点 [/h2]

[h2] 核心功能 [/h2]
[b] LLM 客户端 [/b]
兼容 OpenAI / DeepSeek / 本地 Ollama 等所有 OpenAI Chat Completions 格式的 API。支持 JSON 强制模式（response_format: json_object），本地模型可关闭。

[b] 异步请求队列 [/b]
所有 AI 请求在后台线程执行，不阻塞游戏主线程。每个请求独立冷却，过期请求自动丢弃。

[b] 上下文构建 [/b]
自动采集游戏状态并打包为文本供各模块使用：
[list]
[*] [b]地图上下文[/b]：时间、殖民者、食物、威胁、季节、天气
[*] [b]小人上下文[/b]：年龄、背景、心情、健康、技能、装备、工作分配、关系
[/list]

[b] 上下文过滤器 [/b]
通过"上下文过滤"设置页精确控制哪些游戏信息注入 Prompt，节省 Token。提供最小/标准/完整三种预设，也可自定义勾选 28+ 个选项。

[b] 调试工具 [/b]
[list]
[*] [b]AI Debug Log[/b]：浮动窗口，查看每次 AI 调用的完整 Prompt + Response
[*] [b]请求悬浮窗[/b]：右下角实时显示 AI 请求状态
[*] [b]Dev 菜单[/b]：测试连接、查看上下文、清除冷却等
[/list]

[h2] 常见问题 [/h2]
[b] Q: 支持哪些大模型？ [/b]
A: 任何兼容 OpenAI Chat Completions API 的模型均可使用，包括 OpenAI GPT 系列、DeepSeek、本地 Ollama 等。

[b] Q: 会不会影响游戏帧率？ [/b]
A: 不会。所有 AI 请求在后台线程执行，主线程只处理回调结果。

[b] Q: API Key 安全吗？ [/b]
A: API Key 仅存储在本地 RimWorld 设置文件中；发起 AI 请求时会发送给你配置的 API 端点，不会上传到 RimMind 自有服务器。

[b] Q: 不填 API Key 会怎样？ [/b]
A: Core 本身不会报错，但所有依赖 AI 的子模组功能将无法工作。

[b] Q: 推荐用什么模型？ [/b]
A: 推荐使用 gpt-4o-mini 或 deepseek-chat，性价比高且响应速度快。本地 Ollama 用户可使用 qwen2.5:7b 等模型。
